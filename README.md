# Text to HTML Converter

A Python GUI application (PyQt5) that converts a plain text file into HTML and sends the result to a C program via a System V message queue. The C program validates the content using a POSIX regex and writes it to an output file.

---

## Project Structure

```
.
├── main.py            # Python application (PyQt5 + sysv_ipc)
├── interfata.ui       # UI file created with Qt Designer
├── validator.c        # C program: receiver + regex validation + file output
└── output.html        # HTML file generated by the C program after validation
```

---

## Requirements

### Python
- Python 3.x
- PyQt5
- sysv-ipc

Install dependencies:
```bash
pip install PyQt5 sysv-ipc
```

> **Note:** `sysv_ipc` is only available on **UNIX/Linux** systems. It does not work on Windows.

### C
- GCC

---

## Build & Run

### 1. Compile the C program

```bash
gcc validator.c -o validator
```

### 2. Start the C program (the receiver)

```bash
./validator
```

The C program will block and wait for a message on the message queue with key `1234`.

### 3. Start the Python application

```bash
python3 main.py
```

---

## How It Works

### GUI Application (main.py)

1. **Browse** — opens a file dialog and fills in the file path field
2. **Convert to HTML** — reads the selected text file and converts it to HTML:
   - The first line becomes an `<h1>` tag
   - Each subsequent non-empty line becomes a `<p>` tag
   - The result is displayed in the text area
3. **Send to C program** — sends the HTML content from the text area through a System V message queue (key `1234`) to the C program

### C Program (validator.c)

- Connects to the message queue with key `1234`
- Reads the message sent by Python
- Validates that the message contains an `<h1>` tag using a POSIX regex
- If validation passes: writes the content to `output.html`
- On exit: destroys the message queue (`IPC_RMID`)

---

## Example

Input text file (`example.txt`):
```
Document Title
First paragraph of the document.
Second paragraph.
```

HTML generated and sent to the C program:
```html
<h1>Document Title</h1>
<p>First paragraph of the document.</p>
<p>Second paragraph.</p>
```

---

## Execution Order

> **Important:** the C program must be started **before** clicking "Send to C program" in the Python app. Otherwise the message queue will not be ready to receive.

Correct order:
```
1. ./validator        ← blocks and waits for a message
2. python3 main.py    ← opens the GUI
3. (select file → Convert to HTML → Send to C program)
```
